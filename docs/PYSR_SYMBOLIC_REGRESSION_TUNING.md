## 推荐正文（可直接替换原小节）

**符号回归参数调优实践**

物理实验数据通常规模不大（数十至数百个采样点）、且含有一定测量噪声，符号回归需要在**拟合精度**、**公式简洁性**与**计算时间**之间折中。本平台基于 **PySR**（底层为 Julia 的遗传规划引擎），将可调参数暴露给教学场景，并在服务端约定一组**与平台实现一致的默认配置**，便于复现与对比。

**（1）搜索强度：`niterations` 与 `population_size`**

迭代次数与种群规模决定遗传算法对表达式空间的探索程度。取值过小容易过早收敛到局部较优式；过大则单次任务耗时显著增加。平台默认采用 `niterations = 100`、`population_size = 20` 作为通用起点；用户可根据数据难度与可接受的等待时间，在界面或请求参数中**上调**上述两项，以换取更充分的搜索。

**（2）表达式复杂度：`maxsize` 与算子集合**

`maxsize` 限制语法树规模，抑制无物理含义的高阶嵌套；`binary_operators`、`unary_operators` 划定允许的运算（如 `+ - * /` 与 `exp、log、sin、cos` 等）。通过收窄算子集或降低 `maxsize`，可引导结果偏向更简单、可解释的公式，适合课堂展示；放宽则有利于拟合强非线性关系，但需警惕过拟合与噪声放大。

**（3）复杂度偏好：`complexity_of_operators`**

PySR 通过为不同算子赋予**复杂度权重**，在帕累托前沿上平衡「误差—复杂度」。平台在默认配置中为常用算子设置了基础权重；用户可在高级参数中调整，使模型在同等误差下更倾向选择结构更简单的候选式，从而在结果列表中获得更符合「奥卡姆剃刀」教学目标的排序，而**无需**在应用层改写遗传算法的适应度函数。

**（4）调参策略建议**

在固定噪声水平与变量维度的前提下，建议采用**由弱到强**的策略：先使用默认参数观察方程族与损失；若残差偏大或形式过于复杂，再逐步提高 `niterations`、`population_size`，或按需微调 `maxsize` 与算子集合。上述过程强调**可解释、可复现**，具体数值改进幅度因数据集而异，宜在结题材料中以**案例描述**或**附录实验记录**呈现，而不宜使用未定义度量下的笼统准确率百分比。
